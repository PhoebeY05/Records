Formatting Notes

    {% if site_mode == "shows" %}
  • MAL XML export: upload the animelist_*.xml file generated by MyAnimeList's Export page. All anime are imported in one upload; the my_status of each entry decides which table it lands in. Episode progress, MAL score, your MAL comments, and the source MAL URL are saved as notes; a score of 9+ is marked as a favorite. The same XML is persisted server-side so genre inference can hit https://myanimelist.net/anime/<series_animedb_id> directly instead of re-searching. When series detection is enabled, the franchise list at https://anime.plus/<mal_username>/entries?sender=franchises&media=A is pulled in a single request and all member shows are grouped under the franchise's root title (e.g. Franchise: Bleach). For shows anime.plus does not cover, each MAL page's full Related Entries section (Prequel, Sequel, Side Story, Spin-off, Alternative Version, etc.) is scraped as a fallback. Fallback lookups are cached in mal_relations_cache.json.
    • {% endif %}
  • Completed: title plus finish date in brackets, for example: Harry Potter and the Philosopher's Stone（17 Jan 2019 end)
  • {{ site_tbr_label }}:
    • Rich documents (Pages, Word, etc.): use your MAL export for show records; the XML status decides the table automatically, so there is no separate extra-status option here.
    • Plain text: use section lines; show imports route by status automatically, with no extra-status bucket in this mode.
    • Metadata markers like #nice, ✔, and series markers like }98 still work on {{ site_record_label|lower }} lines.
  • Unfinished: title is everything before the first progress marker, and notes start at that marker. Supported markers include 章, 页, Pg, page, chapter, Chapter, volume, Volume, plus forms like 14章, 第661章, 12页, and 491章/ 77章. The helper text "(haven't read)" is stripped out on import if you include it beside the title.
    • {% if site_mode == "shows" %}
  • Duplicate handling: MAL XML entries are deduplicated by title and date, and the latest matching entry wins. Show mode does not use a separate extra-status bucket.
    • {% else %}
  • Duplicate handling: Completed ignores exact duplicate entries with the same title and date, adds 1 reread for the same title on a different date, and ignores any extra read-count text on duplicate lines; if the notes mention 番外 without 看不到, the status is set to Left Extras. {{ site_tbr_label }} keeps earliest in file, Unfinished keeps latest in file.
    • {% endif %}
